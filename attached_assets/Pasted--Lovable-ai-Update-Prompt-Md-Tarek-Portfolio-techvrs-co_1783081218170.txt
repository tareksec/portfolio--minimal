## 1. Light / Dark Mode
Add a theme switcher icon (sun/moon) in the navbar, right side, next to the last nav link.
- **Default theme on page load = Light mode.** This is not optional — when a new visitor lands on the site with no prior selection, they must see the light theme first, not dark.
- **Light mode must be minimalist and clean:**
  - Background: pure white or very light off-white (e.g. #FFFFFF or #FAFAFA) — no dark panels, no gradient washes.
  - Text: dark navy/near-black for high contrast and readability.
  - Accent color (green/cyan) used ONLY on small elements — buttons, links, icons, badges, active nav state, card borders on hover. Never as a background fill or glow behind large sections.
  - Cards: white background with a thin light-gray border and soft subtle shadow — no glassmorphism/blur needed in light mode.
  - Hero background animation (dots/particles): extremely subtle, almost invisible (5–10% opacity), using light gray or faint accent color — it should feel like texture, not a visual effect.
- **Dark mode must also be minimalist, not neon/cyberpunk:**
  - Background: deep navy or near-black (e.g. #0B0F14).
  - Same restrained accent usage as light mode — accent color only on interactive elements, not as ambient glow across the page.
  - Aurora/particle background animation: low opacity, calm — a "clean dark UI" feeling, not a glowing sci-fi effect.
  - Cards: dark surface with a subtle lighter border, soft shadow — glass/blur effect is fine here but should stay subtle, not heavy.
- Both themes should read as the same professional design system, just inverted in color — same spacing, same typography scale, same restraint. No neon-heavy, glowing, or "gamer" aesthetic in either mode.
- Persist the user's manual toggle choice using React state for the session (no localStorage).
- Smooth 300ms transition on toggle — no flash or instant snap.
